# 功能细节说明：公开前台 × 管理端（v1）

> **已整合**：当前以 **`requirements-master-v1.md`** 为需求主文档；本文保留过程稿细节供对照。  
> **项目定位**：个人学习向，**不**要求与旧实现兼容；见 Master 文档「项目定位」。

> 依据产品需求讨论整理：单站、首期 AI 行业、全站公开免登录、仅管理端登录；第三方 API 经统一出口拉取；**文章正文**不用大模型写摘要；**前台「热门」默认由 AI 约每三天生成一次**（清单落库；实现为服务端 `IntervalTrigger(days=3)`，趋势页另约每 3 天自动拉取）；灵感仅管理员手动点击且可用大模型；灵感有版本历史；异动算法提醒；板块细分；来源标注规则区分应用/工具/大模型等。

**公开前台范围（修订）**：访客 **看趋势**；在 **资源** 中阅读 **文章**；并可访问 **网站介绍与免责声明** 独立页面。**灵感不对用户展示**，全部放在 **管理端**（生成、查看、历史）。

## 一、信息架构总览

| 区域 | 访问方式 | 主要目的 |
|------|----------|----------|
| **公开前台** | 无需登录 | **趋势**；**资源**（文章）；**网站介绍与免责声明**（独立页） |
| **管理端** | 管理员登录 | 连接器、镜像出口配置、数据同步、异动与提醒、**灵感**（仅此处）、**热门定时任务**查看/手动重跑、文章/资源维护、自营条目、消耗统计 |

## 二、公开前台 — 功能细节

### 2.1 全局

- **主导航**：**趋势**、**资源**、**关于/本站**（命名可自定，指向 **2.6** 独立页）；行业（首期固定 AI，预留切换；未开放行业置灰「即将上线」）。
- **页脚**：短文案 + 链向 **网站介绍与免责声明** 全文页（见 **2.6**）；**不**提供「灵感」入口；灵感 **仅存在于管理端**。
- **语言**：默认中文；专有名词可保留英文并附简短说明（可后续扩展 i18n）。
- **无登录**：不展示登录入口；无用户中心、无投稿入口。

### 2.2 首页 / 总览

- **行业上下文**：展示当前行业名称（AI）及一句范围说明。
- **快捷入口**：仅 **趋势**、**资源（文章）** 两个入口（或首页即嵌入两模块摘要 + 「查看更多」）。
- **默认「热门」摘要（与 2.5 联动）**  
  - 首页若展示两模块预览：默认各展示 **当前生效 AI 热门清单** 中的若干条（数量 **N** 可配置，建议 5～10）。  
  - **趋势侧 / 资源侧** 的「热门」**默认**来自 **约每三天一次** 的 **AI 生成结果**（见 **2.5**），非实时调模型。  
  - 提供 **「进入趋势」/「进入资源」** 或 **「查看更多」** 进入完整页。

- **可选 KPI 条**：如「覆盖连接器数」「最近数据更新时间」（来自元数据，非模型生成）。

### 2.3 趋势仪表盘（核心）

- **默认展示：当前 AI 热门快照**  
  - 用户 **进入趋势页**（或首页趋势区）时，**默认**突出展示 **2.5 最近一次定时生成** 中为趋势区生成的 **热门若干项**（如 Top **N** 指标/板块/图）。  
  - **兜底**：若定时任务失败或尚无快照，可回退为 **纯规则**（如异动 Top、运营白名单）或 **上一版快照**（可配置策略）。  
  - 提供 **「查看全部趋势」** 展开完整时间范围与全部分区/图表；切换时间、板块后，可按 **非 AI** 的常规图表逻辑浏览（与热门快照独立）。

- **时间范围**  
  - 预设：7 天 / 30 天 / 90 天 / 本月 / 本季度（可裁剪）；**默认时间窗** 与热门计算一致（如默认近 7 天或近 30 天，可配置）。  
  - 自定义：起止日期；**同一页所有图表共用同一时间窗口**（切换后全局刷新）。  
  - 无数据：空态文案 + 提示「部分连接器不可用」若适用。

- **板块维度**  
  - 用户可选 **板块**（与后台配置的 AI 子类一致，如：大模型、应用、工具、框架等）。  
  - 「全部」表示跨板块聚合视图（定义需在指标字典中说明聚合方式）。**默认**可先展示 **全站热门** 再允许收窄板块。

- **图表类型（示例清单，可配置）**  
  - 时间序列折线/面积：单指标或多系列对比。  
  - 柱状/条形：Top N、板块对比。  
  - 堆叠/占比：构成分析。  
  - **所有序列**可下钻到「来源说明」（简要：提供方、更新时间；可与资源规则一致）。

- **异动提示（只读）**  
  - 公开侧 **不强制**展示内部异动等级；可选展示「近期波动较大」类 **非敏感**标签（若产品希望弱化，可仅管理端可见）。  
  - *默认建议*：前台弱化异动算法细节，避免误导；重点在数据与图表。

### 2.4 资源 · 文章（非模型摘要）

- **定位**：资源区以 **文章** 为呈现单位（列表 → 详情页），内容由 **连接器同步 + 结构化/模板/截取** 入库，或运营编辑；**不使用大模型生成文章摘要**（口径不变）。

- **默认展示：当前 AI 热门快照**  
  - 用户 **进入资源列表**（或首页资源区）时，**默认**文章顺序与 Top 展示 **来自 2.5 定时 AI 生成的热门清单**（Top **N** 可配置）。  
  - **兜底**：任务失败或无快照时，回退 **最新优先** 或 **运营精选**（可配置）。  
  - 提供 **「查看全部文章」** 进入完整列表；用户再使用筛选/排序。

- **列表与挑选（筛选）**  
  - **板块**：必选或可选筛选项，与后台配置的 AI 子类一致（如大模型、应用、工具等）；支持「全部」。  
  - **时间与实效**：按 **发布时间/入库时间**、或文章所涉 **数据时间范围** 筛选（具体字段在内容模型中定义）。  
  - **实际情况维度**（可与运营配置挂钩）：如 **内容来源类型**（第三方同步 / 自营维护）、**类型标签**（第三方衍生 / 自营工具 / 自营大模型 / 应用 等）、**是否精选/置顶**——用于在真实入库内容上「挑选」读者看到的文章集合。  
  - **排序**：在 **「查看全部」** 或筛选结果中，可选 **最新优先**、**按板块内推荐**；**默认首屏热门**仍以 **2.5 定时快照** 为准。

- **列表/卡片要素**：标题、摘要一行（来自字段或截取）、时间、板块、类型标签。

- **详情页**  
  - **正文**：结构化字段 + 模板 + 截取/抽取；不冒充「AI 长文生成」。  
  - **展示规则**（与来源标注策略一致）  
    - **第三方衍生 / 应用类文章**：**强制**体现第三方来源（提供方、获取时间、可公开链接）。  
    - **自营工具 / 大模型**：可不标第三方来源；标明「自营维护」或等价说明。  

- **与「灵感」的关系**：资源文章 **不包含** 管理端「灵感」产物；二者在数据与页面上 **隔离**，访客 **看不到灵感**。

### 2.5 热门清单：AI 定时快照（默认，约每三天）

- **机制**  
  - **默认**：前台「热门」由系统 **约每三天执行一次** 定时任务，调用 **大模型**，基于 **真实聚合数据与文章元数据**，生成 **趋势热门建议 + 资源文章热门排序**，并 **落库为快照**（含版本号、生成时间、适用周期说明）。  
  - **频率**：**约每 3 天至少一次**（实现为 `IntervalTrigger(days=3)`，可再调参）；两次生成之间，前台 **只读该快照**，**不**对每位访客实时调大模型。  
  - **输入约束**：仅投喂 **已入库的统计摘要、指标键、板块、文章标题/标签等**；**禁止**编造不存在的数值；数据不足时 **降级**（减少条数或明确「数据不足」）。  
  - **输出**：  
    - **趋势**：本周期建议优先展示的 **指标/板块/图表项** 列表（ID 可映射到已有图表配置）。  
    - **资源**：本周期 **文章 ID 排序** 或 Top **N** 文章列表。  
  - **展示**：访客在趋势/资源默认视图使用 **当前生效快照**；可展示 **「热门更新于 yyyy-mm-dd」**（可选）。

- **与文章正文的区别**：定时热门任务 **不**生成文章正文；正文仍为非大模型管道（见 2.4「定位」）。

### 2.6 网站介绍与免责声明（独立页面，必做）

- **路由**：公开可访问的独立 URL（如 `/about` 或 `/legal`，产品定名）；**无需登录**。  
- **页面结构（建议分节，长文可锚点导航）**  
  1. **网站介绍**：站点定位与范围（如 AI 行业趋势与资源聚合）、运营主体说明（若适用）、**非**投资建议/非法律意见等定性表述。  
  2. **数据与来源**：数据主要来自 **第三方开放 API**；经统一出口拉取；**热门**由 **AI 约每三天生成** 的推荐快照驱动（可简述机制，避免用户误解为实时人工编辑推荐）。  
  3. **免责声明（全文）**：包括但不限于：信息仅供参考、不保证完整准确及时、用户自行判断风险、第三方服务条款链接提示、转载与引用规范等（**文案需法务审定**）。  
  4. **可选**：联系方式、隐私政策入口（若后续单页）、**文档最后更新日期**。  
- **与其他页的关系**：趋势、资源页 **页脚**保留一句摘要 + **「查看完整说明」** 链到本页；避免仅在页脚一行字替代完整免责。  
- **管理端（建议）**：上述正文支持 **后台可编辑**（富文本或 Markdown）与 **版本记录**；重大变更需审核后发布（流程自定）。

### 2.6a 趋势/资源页内补充（非独立页）

- 在 **趋势页、资源列表/详情** 可保留 **折叠区**：主要数据提供方名称、「数据更新于」；不展示密钥或内部 endpoint；详细说明 **以 2.6 独立页为准**。

### 2.7 公开前台 — 非功能（体验）

- 首屏加载目标（与后端约定 SLA，如 P95 &lt; 3s 可填数值）。  
- 弱网：骨架屏或渐进加载图表。

## 三、管理端 — 功能细节

### 3.1 登录与账号

- **登录**：用户名 + 密码；失败锁定与策略读取服务端全局安全策略（最小长度、失败次数等）。  
- **会话**：登出；过期重新登录。  
- **角色（建议）**  
  - **viewer**：只读管理数据、不可改连接器、不可生成灵感。  
  - **operator**：维护连接器、查看异动与提醒、可触发生成灵感（若你合并职能可删减）。  
  - **admin**：全量 + 用户与预算配置。  
  - *若首期仅一人，可只实现 admin，角色预留。*

### 3.2 总览（管理首页）

- 连接器健康：成功率、最近同步时间、错误次数。  
- 今日/本周 **大模型调用次数与 token**（按场景汇总，见 3.9）。  
- 待处理提醒数：异动 L1/L2、建议复核灵感数（若有）；**热门定时任务**失败/未执行告警（若有）。  
- 快捷入口：连接器、异动、灵感生成、**热门快照**（最近一次生成结果与手动重跑入口）。

### 3.3 镜像出口 / 统一拉数配置

- **配置项**：基础 URL 或路由前缀、超时、重试策略、是否启用（开关）。  
- **说明**：需求层「所有第三方 API 经统一出口」；实现可为网关/反向代理。  
- **审计**：关键变更写入审计日志（操作者、时间、前后摘要）。

### 3.4 连接器（仅 API，首期）

- **列表**：名称、提供方、行业/板块绑定、启用状态、最近同步、失败原因摘要。  
- **新建/编辑**  
  - 类型：`api`（首期唯一）；预留 `crawler` 禁用或隐藏。  
  - 认证：无 / API Key / 其他（不落库到前端展示；仅存服务端）。  
  - 端点路径、请求方法、参数模板、分页规则（按连接器文档）。  
  - 同步频率：最小间隔（遵守免费 API 限额）。  
  - **字段映射**：原始 JSON → 我方结构化字段（标题、时间、数值、标签等）。  
- **手动同步**：立即拉一次（受频控）。  
- **测试连接**：返回 HTTP 状态与样例字段（脱敏）。

### 3.5 指标与板块配置

- **板块**：增删改、排序、是否在前台展示。  
- **指标字典**（与图表绑定）  
  - 指标键、中文名、单位、聚合方式（sum/mean/count 等）、适用板块、数据来源连接器。  
  - 是否参与异动监测、用原始值还是日增量（锁死语义防误用）。  
- **前台「热门」策略（与公开 2.5 对应）**  
  - **默认**：由 **AI 约每三天生成一次** 热门快照（趋势推荐项 + 资源文章排序），见 **2.5**；**约每 3 天至少执行一次**（调度可配置）。  
  - **配置项**：执行时刻、所用模型/提示版本、Top **N**、喂给模型的 **输入摘要条数上限**、**失败兜底**（沿用上期快照 / 纯规则异动 Top / 资源按最新或精选）。  
  - **管理端能力**：查看 **最近一次成功时间**、当前生效快照概要、历史版本；**手动立即重跑**（admin/operator，**频控**防止滥用）；失败时 **告警**（邮件/站内可选）。  
  - **可选补充**：运营 **白名单/黑名单**（强制包含或排除某些指标键、文章 ID），在定时快照结果基础上合并（规则优先或人工覆盖策略需明确）。

### 3.6 文章 / 我方资源生成（无大模型）

- **生成任务**：按连接器增量拉取 → 映射结构化字段 → 模板生成展示用短句（若有）→ 截取/抽取（若有）→ 入库为 **文章** 资源。  
- **人工覆盖**：运营可编辑标题、正文片段、板块归属、是否精选、上线/下线（仍不调用大模型写摘要）。  
- **类型标注**：第三方衍生 / 应用（强制来源）/ 自营工具 / 自营大模型 —— **校验**应用类必须填来源字段。  
- **前台可见性**：仅 **已发布** 且符合筛选规则的文章出现在公开 **资源** 列表；与 **灵感** 库无交集。

### 3.7 异动监测与提醒

- **算法（默认）**  
  - 监测单元：行业 × 板块 × 指标（可细到连接器）。  
  - 特征：稳健 Z、短期斜率变化、星期对齐残差等合成 **异动分数 S**；**L1/L2 分级**与 **冷却期**（避免重复提醒）。  
  - 板块级：当多监测单元同时超阈值，升成 **板块级提醒**。

- **管理端界面**  
  - **异动列表**：时间、单元、分数、等级、板块、是否已读。  
  - **异动详情**：曲线快照链接、阈值参数只读展示、关联指标说明。  
  - **规则配置**（admin）：窗口长度、L1/L2 阈值、冷却时间、板块聚合参数 K 等。

- **与灵感联动**  
  - 某灵感绑定的时间段之后，若同板块出现 **L1** 或板块级强提醒，该灵感在管理端显示 **「环境可能已变化」** 徽标。  
  - **不自动**更新灵感正文。

### 3.8 灵感（大模型 · 手动 · 历史 · 仅管理端）

- **不对用户展示**：访客 **不能** 在前台看到灵感（无列表、无详情、无入口）。灵感 **全部** 在管理端完成：生成、编辑、历史与版本查询。  
- **与资源文章区分**：灵感为运营/策略内化用草稿与复盘；**资源区仅展示「文章」**，二者数据隔离。

- **入口条件**：仅管理员（或 operator，按角色定）可见「灵感工作台」。

- **生成流程**  
  1. 选择 **板块**、**数据时间范围**、可选 **关联异动事件**（从列表勾选）。  
  2. 展示 **将送入模型的上下文摘要**：聚合后的指标要点、我方资源要点、**不含密钥**；允许管理员删减片段。  
  3. **手动点击「生成」** → 调用大模型 → 返回草稿。  
  4. 管理员可编辑标题/正文/标签 → **发布** 或 **存为草稿**。  
  5. 每次发布产生 **新版本**；旧版本保留。

- **历史与查询**  
  - 列表筛选：板块、时间、关键词、版本号、操作者、是否当前发布。  
  - 详情：任意历史版本只读查看；**版本对比**（相邻版本 diff 或摘要差异）。  
  - **发布状态**：若未来恢复对外灵感频道，可再启用「对外发布版本」；**当前需求下**灵感仅内部留存与复盘，无公开 URL。

- **提醒栏**  
  - 来自异动的「建议复核灵感」队列：列出受影响灵感卡片 + 快捷进入重新生成流程。

### 3.9 大模型消耗统计

- **每次调用记录**：时间、场景（固定枚举，至少包含）：  
  - `inspiration_generate`：管理员手动灵感；关联灵感版本 ID；**操作者**为用户账号。  
  - `hot_rank_weekly`（历史命名，**含**定时热门快照）：**热门定时重建**；**操作者**为 `system` 或留空；关联快照版本 ID。  
  - `hot_rank_manual`：管理员 **手动重跑** 热门；操作者为实际账号。  
  - 模型名、输入/输出 token（或计费单位）、成功/失败。  
- **报表**：按日/周/月；按场景；按模型；可选按操作者。**定时热门任务**可单独看 **固定周期消耗**，便于预估成本。  
- **预算**：日/月阈值、超额告警、可选熔断（例如：暂停 **手动灵感** 或 **手动重跑热门**，**不建议**长期静默跳过定时任务以免前台长期无快照——可改为告警 + 沿用兜底）。

### 3.10 自营条目维护

- **AI 工具 / 大模型**：表单维护名称、简介、链接等；**不强制**第三方来源字段。  
- **AI 应用**：表单 **强制**来源字段校验（与公开规则一致）。

### 3.11 审计与安全

- 审计日志：登录、连接器变更、规则变更、灵感发布/下线、**热门快照生成/手动重跑**、**网站介绍/免责页发布**、预算变更。  
- 密钥：仅服务端；管理界面永不显示明文 Key，仅掩码与「已配置」状态。

### 3.12 公开页文案：网站介绍与免责声明

- **编辑**：管理员在后台维护 **2.6** 页面正文（分段：介绍、数据说明、免责全文等）；支持预览、保存草稿、发布。  
- **版本**：建议保留历史版本，便于回溯与合规审计。  
- **非大模型**：该页文案 **默认**人工维护；不要求 AI 生成免责条款（法务审定为准）。

### 3.13 管理端 — 非功能

- 强认证；管理 API 与公开 API 分离；禁止匿名触发生成类接口。  
- 列表分页、导出 CSV（可选）：消耗报表、审计日志。

## 五、与后续迭代的衔接

- **若未来要对用户开放灵感**：需单独立项与发布开关；**当前默认**为前台 **永不展示** 灵感。  
- **爬虫连接器**：管理端连接器类型扩展 `crawler`，单独评审后再开放配置 UI。  
- **多行业**：前台行业切换 + 后台行业包（连接器包、板块模板、指标包）。  
- **本地开源模型**：可作为灵感后端实现选项，不改变「手动点击 + 计量」需求。
